### Internationalization (i18n)

Back to README: [../README.md](../README.md)

Flutter gen-l10n is enabled and configured via `l10n.yaml` with ARB files under `lib/l10n/`.

```1:4:l10n.yaml
arb-dir: lib/l10n
template-arb-file: app_en.arb
output-localization-file: app_localizations.dart
```

Generated localization delegates are wired in `lib/main.dart`:

```37:41:lib/main.dart
locale: value.locale,
localizationsDelegates: AppLocalizations.localizationsDelegates,
supportedLocales: AppLocalizations.supportedLocales,
```

### Adding a new locale

1. Create `lib/l10n/app_<locale>.arb`, e.g., `app_es.arb`.
2. Add translated key-value pairs matching `app_en.arb`.
3. Run `flutter pub get` (gen-l10n runs on build).
4. Use `AppLocalizations.of(context)!.<key>` in widgets.

ARB examples: `lib/l10n/app_en.arb`, `lib/l10n/app_nb.arb` (Norwegian).

### Formatting

- Use `intl` package APIs for dates and numbers (`intl: any` in `pubspec.yaml`).
